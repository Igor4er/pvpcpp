# pvp++

## Загальна структура файлів, або як менеджити проєкти:
```
.
├── 0 <- Вейпоінт
│   ├── 1 <- завдання
│   │   ├── 1
│   │   ├── 1.cpp <- Головний файл має назву НОМЕР_ЗАВДАННЯ.cpp
│   │   └── a.out <- Файл об'єкта, можна його ігнорувати
│   ├── 2
│   │   ├── 2 <- Номер завдання без розширення це виконувальний файл, для тих хто на вінді може бути розширення .exe
│   │   ├── 2clang <- Це теж вик. Файл, просто скомпільвонай не gcc a clang
│   │   ├── 2.cpp
│   │   └── a.out
│   └── build <- Це якись треш який VSCode створив, можна ігнорувати
│       └── Debug
└── README.md
```
